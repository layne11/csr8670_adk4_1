typedef struct __GANCS GANCS

The apple notification internal structure for the client role.

This structure is visible to the application as it is responsible for managing resource to pass to the apple notification library. The elements of this structure are ONLY modified by the apple notificaiton library. The application SHOULD NOT modify the values at any time as it could lead to undefined behaviour.

gatt_ancs_status_t GattAncsInit ( GANCS *  ancs,
Task  app_task,
uint16  cid,
uint16  start_handle,
uint16  end_handle 
)

After the VM application has used the GATT manager library to establish a connection to a discovered BLE device in the Client role, it can discover any supported services in which it has an interest. It should then register with the relevant client service library (passing the relevant CID and handles to the service). For the apple notification client it will use this API. The GATT manager will then route notifications and indications to the correct instance of the client service library for the CID.

Parameters
ancsA valid area of memory that the service library can use.
app_taskThe Task that will receive the messages sent from this apple notification client library.
cidThe connection ID.
start_handleThe start handle of the apple notification client instance.
end_handleThe end handle of the apple notification client instance.
Returns
The status result of calling the API.
void GattAncsSetNSNotificationEnableRequest ( const GANCS *  ancs,
bool  notifications_enable,
uint16  notification_mask 
)

This API is used to write the client characteristic configuration on a remote device, to enable notifications for Notification Source Characteristic with the server. An error will be returned if the server does not support notifications.

Parameters
ancsThe client instance that was passed into the GattAncsInit API.
notifications_enableSet to TRUE to enable notifications on the server, FALSE to disable them.
notification_maskThis mask is used to set the notification which the application is interested to either mask/unmask. Application can make use of the above defined category macros to set/clear each category. For example: notification_mask = ANCS_INCOMING_CALL_CATEGORY | ANCS_EMAIL_CATEGORY...
Returns
A GATT_ANCS_SET_NS_NOTIFICATION_CFM message will be sent to the registered application Task.

gatt_ancs_status_t GattAncsDestroy ( GANCS *  ancs)

When a GATT connection is removed, the application must remove all client service instances that were associated with the connection (using the CID value). This is the clean up routine as a result of calling the GattAncsInit API. That is, the GattAncsInit API is called when a connection is made, and the GattAncsDestroy is called when the connection is removed.

Parameters
ancsThe client instance that was passed into the GattAncsInit API.
Returns
The status result of calling the API.
