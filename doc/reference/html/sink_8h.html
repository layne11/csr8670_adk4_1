Attempt to claim the indicated number of extra bytes in a sink.

Parameters
sinkThe sink to claim.
extraThe number of bytes to attempt to claim.
Returns
The offset of the claimed region if the claim was successful, 0xFFFF otherwise.

Claims will certainly fail if the sink is invalid, or if SinkSlack indicates that the space is unavailable.

Note
This call will return zero if the sink stream is connected to another stream.
This trap will block VM application if there are not sufficient buffer memory resources to support the claim.
uint8* SinkMap ( Sink  sink)

Map the sink into the address map, returning a pointer to the first byte in the sink.

Parameters
sinkThe sink to map into the address map.
Returns
zero if the sink is invalid.

Only the total number of claimed bytes (as returned by SinkClaim(sink,0)) are accessible. At most one sink can be mapped in at any time; pointers previously obtained from SinkMap become invalid when another call to SinkMap is made.

Note
This call will return zero if the sink stream is connected to another stream.

Flush the indicated number of bytes out of the sink.

Parameters
sinkThe Sink to flush.
amountThe number of bytes to flush.

The specified bytes of data are passed to the corresponding byte stream, for example out to the UART, or into BlueStack as if sent by a RFC_DATA_IND for UART/RFCOMM sinks respectively.

Returns
TRUE on success, or FALSE if the operation failed because the sink was invalid or amount exceeded the size of the sink as reported by SinkClaim(sink, 0).

This call will return FALSE if the sink stream is connected to another stream.

bool SinkClose ( Sink  sink)

Request to close the sink.

Parameters
sinkThe sink to close
Returns
TRUE if the source could be closed, and FALSE otherwise.

Sink stream can be closed only if the requested sink stream is not connected to any stream. If the requested stream is connected, then TransformDisconnect or StreamDisconnect should be called before SinkClose.

Some sinks, such as RFCOMM connections or the USB hardware, have a lifetime defined by other means, and cannot be closed using this call.

bool SinkPollAwayTime ( Sink  sink,
uint16 *  msec 
)

Read the away time on the underlying ACL.

Parameters
sinkidentifies the underlying ACL
msecreceives the away time if the call succeeds (unmodified otherwise)
Returns
TRUE if the sink identifies an ACL and the away time on that link could be read, FALSE otherwise.

The away time is the time since any packet was received on that ACL and is reported in milliseconds. If the time exceeds 0xFFFF, 0xFFFF will be returned (this is unlikely with sensible link supervision timeouts.)
